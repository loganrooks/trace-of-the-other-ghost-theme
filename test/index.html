Technical Notes

  • File Protocol: These tests can be run locally using file:// protocol
  • Console Logging: Open browser developer tools to see detailed logs
  • JavaScript Errors: Any errors indicate implementation issues
  • Performance: Monitor performance tab during testing
  • Cross-Browser: Test in Chrome, Firefox, Safari, and Edge

⚠️ Expected Behaviors

Some "failures" are philosophical features:

  • Deferred loading may occasionally refuse to load (différance at work)
  • Marginalia may become more prominent than main text (supplement mechanics)
  • Navigation may lead to unexpected places (pharmakon effects)
  • Text may fade, glitch, or revise itself (temporal effects)
  • Print styles will dramatically alter the appearance

These are not bugs - they are features performing philosophical concepts.

Ghost CMS Integration Testing

To test with actual Ghost CMS:

  1. Install Ghost locally or use Ghost(Pro)
  2. Upload theme files to Ghost's content/themes directory
  3. Activate the "trace-of-the-other" theme
  4. Create test posts with marginal data attributes
  5. Verify all Handlebars templates render correctly
  6. Test theme settings and customization options

Sample Ghost post content for testing:

<p data-margin-left="Critical voice" data-margin-right="Translation note">
    Main text content
</p>

<div class="palimpsest" data-erased="Previous version" data-erased-2="Earlier draft">
    Final version text
</div>

<div class="temporal-content">
    Content with temporal effects
</div>
